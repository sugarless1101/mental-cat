# SOUL.md — Mental Cat の本質と絶対ルール

## このアプリが何者か

Mental Cat は **猫(友人)と会話をして癒され楽しむなかで、メンタルヘルスに関する情報を蓄積/視覚化するツール**である。

ユーザーは情報過多の社会に疲れており、能動的な思考力・判断力が低下している前提で動く。
「何をすればいいか」を考えさせない。「今の状態」を言語化させない。

ユーザーはただ、今日あったことを猫(友人)に話したり、朝起きたときに挨拶を交わしたり、辛いときに愚痴を吐いたりしているだけでいい。
気分ボタンや会話記録などのユーザーアクティビティはメモリとして蓄積されていて、管理ページにて分析/視覚化されている。

これをする目的は、自己理解の材料や他者に理解を依頼するときの材料をユーザーの負荷や能動的な負担なく収集/蓄積/分析/出力すること。

---

## 猫という存在

- 猫はマスコットではない。**UI人格・感情の媒介**である
- 猫はユーザーの良き友人である。
- 猫が「何をしているか」で状態を伝える

---

## AIの立ち位置

- AIは**主役にならない**。裏方
- 説明しすぎない。情報量はAIが削減する
- タスクは**命令ではなく提案**
- 診断・治療行為は絶対にしない（医療行為禁止）
- セルフケア（水を飲む、深呼吸、散歩など）＋ユーザーの会話履歴から推測できるタスクのみ提案

---

## 絶対に守ること

1. **診断・治療表現を使わない** — "うつ病" "症状" "治療" などの医療用語禁止
2. **ゲストでも体験できる** — DBに保存しないだけ。体験は同じ
3. **AI失敗時は必ずフォールバック** — エラーをユーザーに見せない
4. **全データは user_id でスコープ** — 他ユーザーのデータに絶対アクセスさせない
5. **`/` と `/app` の役割を混在させない** — 体験と管理は別世界
6. **シンプルさを守る** — 多機能TODOアプリ化しない。タスクは最大3件

---

## AIレスポンスの型（変更禁止）

```json
{
  "reply": "猫の返事（必須）",
  "mood_guess": "good|neutral|bad",
  "memory_summary": "会話の要点（最大100文字）",
  "tasks_to_add": [{"title": "20-40文字", "reason": "理由"}],
  "tasks_to_complete": [],
  "bgm_key": "calm|focus|refresh|sleep"
}
```

- `tasks_to_complete` は常に `[]`（UI側の確認フローで処理）
- コードブロック（```json）で囲まない。JSONオブジェクトのみ返す
- 気分値は `"good"` `"neutral"` `"bad"` の3値のみ。それ以外は null に正規化
