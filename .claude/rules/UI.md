# UI.md — Mental Cat UI規則

## 最上位原則（必ず守る）

- **すべてを表示しない**
- **ユーザーに考えさせない**
- **選択肢は常に最小限**
- **体験（`/`）と管理（`/app`）を混在させない**
- **AIは主役にならない**
- **「つい押したくなる」「ついやりたくなる」といった体験で、メンタルヘルスの情報を収集するよう工夫する**

---

## UIのこだわり
- 表示する画面はシンプルに
- AIがリコメンドして、ユーザーが受動的に楽しめるコンテンツのみを動的に配置する。
- シンプルなだけではなく、アニメーションの装飾などを用いて程よい情報量を常に渡す。
- 動的なデザインを用いる。

## 画面分離ルール（絶対）

| パス | 役割 | トーン |
|------|------|--------|
| `/` | 体験・没入・癒し | 黒基調・低コントラスト・余白重視 |
| `/app` | 記録・振り返り | 白基調・情報密度高め・演出最小限 |
| `/admin` | LLMOps観測（開発者専用） | このルール対象外 → `.claude/rules/ADMIN.md` 参照 |

- `/` に管理UIを置かない
- `/app` にBGM・猫アニメ・チャット入力を置かない
- `/admin` はユーザー向けではないため UI.md のデザイン規則を適用しない
- Blade / Layout / CSS を完全分離する

---

## `/` 体験ページ

### 構成要素
| 要素 | 配置 | 補足 |
|------|------|------|
| MoodSelector | 初回は中央・選択後は左上に縮小 | 3択（😞😐😊）のみ |
| CatAnimation | 中央 | 状態: idle / calm / happy / sad |
| ChatInput | 画面下部 | 猫への入力、唯一の入力UI |
| TaskWidget | 左側 | 最大3件・AIタスクのみ・手動追加UIなし |
| BGMWidget | 右側 | YouTube埋め込み・気分に応じて切替 |

### BGMマップ（要確認：bgm_key と mood の対応）
```
calm    → 要確認：bgm_key "calm" のYouTube URL
focus   → 要確認：bgm_key "focus" のYouTube URL
refresh → 要確認：bgm_key "refresh" のYouTube URL
sleep   → 要確認：bgm_key "sleep" のYouTube URL
```
（docs/UI.md に good/neutral/bad 別URLあり。AIが返す bgm_key との対応を要確認）

---

## `/app` 管理ページ

### 表示要素
- **Mood Log**: 日時・気分の縦並びリスト（同一1分以内は最新1件に集約）
- **Chat History**: 閲覧のみ・入力欄なし
- **Task管理**: Todo 未完了上位5件 / Done 最近完了分

---

## デザイントーン

### 体験ページ（`/`）
- 黒基調・低コントラスト
- 動き・揺らぎ・余白を重視
- 情報を「見せる」のではなく「感じさせる」
- 現実から切り離す静かな世界観

### 管理ページ（`/app`）
- 白基調・クリーン
- 情報密度は上げてよい
- 演出・アニメーションは不要

---

## 言葉遣い・トーン

- 猫の口調：やわらかく・短く・押しつけがましくない
- タスクの言い方：「〜してみよう」「〜はどう？」など提案形
- エラー時：ユーザーを責めない。猫がうまく話せなかった体で表現
- 医療用語・診断表現は一切使わない
- CAT.mdを参照

---

## やらないこと（非目標）

- 多機能TODOアプリ化
- `/app` でのチャット入力・BGM再生
- 手動タスク追加UI（`/` のTaskWidget）
- ユーザーに選択肢を増やす方向の機能追加
