### [D-9] ダミーメッセージがチャット履歴に残存している 🟢

**症状**
`/` の チャット履歴エリアに「ダミーメッセージ2〜26」という文字列のバブルが表示される。
`/app` の「最近の会話」にも「ダミーメッセージ3〜7」が表示されている。

**原因**
開発中に作成したシードデータ（またはテスト入力）が SQLite の `chat_messages` テーブルに残存している。バグではなく**データの問題**。

**対処**
```bash
# 開発環境のみ（本番では絶対実行しない）
php artisan migrate:fresh
# または tinker で個別削除
php artisan tinker
>>> \App\Models\ChatMessage::where('content', 'like', 'ダミー%')->delete();
```

**影響ファイル**
- なし（DBのデータ問題）

### [D-10] タスク完了マッチングが部分一致しか拾えないケースがある 🟢

**症状**
「散歩してきた」と入力しても「近所の公園をのんびり散歩する」の完了確認が出ない。
AI 経由の応答になってしまう。

**原因**
`findCompletionCandidate()` はタイトルの正規化文字列がユーザー入力を含むか（または含まれるか）の直接マッチを優先し、
Dice 係数による類似マッチはその後のフォールバック。

「散歩してきた」→ 正規化 → `散歩してきた`
「近所の公園をのんびり散歩する」→ stem → `近所の公園をのんびり散歩`

`散歩してきた` が stem に含まれず、stem が `散歩してきた` に含まれないため直接マッチしない。
Dice 係数は閾値（0.4）を下回るため fuzzy マッチも不成立。

**確認済み動作例**
| 入力 | マッチするか |
|------|------------|
| 「音楽を聴いた」 | ✅ `好きな音楽を聴いてリラックス` にマッチ |
| 「散歩してきた」 | ❌ マッチしない → AI 経由 |
| 「お茶を飲んだ」 | ✅ `温かいお茶を淹れて〜` にマッチ |

**対処方針**
現状は許容範囲と判断。もし改善するなら `containsCompletionIntent` の語幹リストに「散歩」「行った」などのキーワードを追加し、タスクタイトルとのトークン重複チェックを導入する。

**影響ファイル**
- `resources/views/pages/landing.blade.php`（`findCompletionCandidate` 関数）

## よくあるトラブルと対処

### TaskWidget が表示されない

**原因**: Vite が起動していない。`md:block` が CSS にコンパイルされない。

```bash
composer dev   # Vite を含む全サービスを起動
```

Windows 環境では `pail`（pcntl 依存）を除いたスクリプトを使用。
`composer.json` の `dev` スクリプトから `"php artisan pail --timeout=0"` を除外済み。

### /admin にアクセスできない（非 local 環境）

`APP_ENV=local` 以外では `X-Admin-Secret` ヘッダーが必要。
`APP_ADMIN_SECRET` が `.env` に設定されていない場合は常に 403 を返す（フェイルクローズド）。

```
curl -H "X-Admin-Secret: your-secret" https://your-app/admin
```

### AI が返答しない / フォールバック表示になる

1. `.env` の `OPENAI_API_KEY` を確認
2. `php artisan queue:listen` が起動しているか確認（`composer dev` で自動起動）
3. `/admin` のLLMログで `ok=false` の件数を確認

### ダッシュボードにアクセスできない（ゲスト）

`/app` はログイン必須。`/login` または `/register` からアカウントを作成する。

### `php artisan migrate:fresh` を実行してしまった

本番・ステージングでは**絶対実行しない**。開発環境でのみ許可。
実行後は `php artisan db:seed` でシードデータを投入する（シーダーがある場合）。
